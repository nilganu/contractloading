SHARED PRICE ROWS (CRITICAL):
- Two periods that have the SAME priceRowGroup in the skeleton share ONE
  visual row of prices in the table. They read prices from the SAME
  (group, board) cell, not from different price rows.
- Example: if skeleton.periods has
    {start: 2026-04-01, end: 2026-06-13, priceRowGroup: "G1"}

    {start: 2026-09-01, end: 2026-09-14, priceRowGroup: "G2"}
  then the rows for the first TWO periods at the same board must have
  IDENTICAL SGL/DBL/TPL/QDP values (taken from the G1 BB / G1 HB cells).
  Only the G2 rows pull from a different row of cells.

DATE / CURRENCY / MEAL PLAN MAPPING (deterministic):
- Use skeleton.currency for Currency and Customer Price Currency.
- Use skeleton.rateType for Rate Type ("Per Person Per Night",
  "Per Person Per Day", or "Per Room Per Night").
- Map board codes: BB → "Bed & Breakfast", HB → "Half Board", FB → "Full
  Board", AI → "All Inclusive", RO → "Room Only".
- Days is a weekday mask "1234567" (1=Mon..7=Sun), NOT a night count.
- Status = "Open" unless contract clearly says "On Request".

CHILD VALUE RULES (UNAMBIGUOUS):
Decide each child column's valueType from the literal cell texts in
skeleton.childPolicies[i].perRoomLiteral:
- If literals contain "%", "free", "FOC", "n/a": valueType =
  "discount_percentage". The stored value is the % OFF the adult rate:
    "free" / "FOC" / "100% off"       →  100
    "-50%" / "50% discount"           →   50
    "-30%"                            →   30
    "n/a" / blank / "-"               →   null   (NEVER 0)
- If literals contain bare amounts ("30 EUR", "€10"): valueType =
  "amount". Store the number as the literal currency amount.
- Build the dynamicChildColumns list once, in skeleton order. Column
  key format: CHD<position#>(<ageFrom>-<ageTo>) when position is set;
  CHD(<ageFrom>-<ageTo>) otherwise.

PER-ROW PER-COLUMN APPLICATION:
For each row's dynamicChildValues:
- Look up skeleton.childPolicies[i].perRoomLiteral[<this row's room name>].
- Apply the rule above. If perRoomLiteral has no entry for this room or
  the literal is n/a / blank / "-", the row's value MUST be null.
- ALSO: if the row's room has skeleton.roomTypes[j].maxChildren == 0,
  ALL dynamicChildValues for that row MUST be null regardless of what
  the child-policy literal text says. A room that doesn't accept children

EXTRA BED (extra adult) FIELD — IMPORTANT:
Leave the "Extra Bed" field as null. The post-processing pipeline reads
skeleton.extraAdultPolicy.perRoomLiteral[<this row's room name>] and
computes the actual currency amount per row using the row's DBL. You
should NOT try to fill Extra Bed yourself — emitting the percentage as a
number will be mis-interpreted as a currency amount.

Only fill "Extra Bed" yourself when:
  - skeleton.extraAdultPolicy is null/missing, AND
  - you can see a clear per-room EUR amount in the contract page (eg

A row whose room's Max Adult equals Min Adult MUST have Extra Bed = null.
